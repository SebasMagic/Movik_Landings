# Movik Landings

QuickPay for carriers and brokers. Static HTML, deployed to Vercel staging today,
bound for `movik.us`.

## How it's built

Three layers. **Only the source layer is edited by hand.**

```
SOURCE (edit this)                 BUILD (generated)        DEPLOY (generated)
<page>-landing/index.html   ─┐
i18n/<page>.es.json          ├──►  <page>-landing/dist/  ──►  deploy/movik-landing/
escape-guide-landing/…       ─┘      *-ENG.html *-ESP.html      /<slug>/  /es/<slug>/
```

English HTML is the source of truth; each `i18n/<page>.es.json` translates it
string by string. The build emits fully static EN and ES pages so crawlers see
real text, never JavaScript-rendered content.

## Commands

```bash
node tools/i18n.js check                    # fail if any string is untranslated
node tools/i18n.js build [page]             # emit ENG + ESP into dist/
node tools/build-escape-guide.js            # build the guide (both languages)
node tools/build-home.js                    # build the homepage hub (both languages)
node tools/deploy-staging.js                # assemble deploy/movik-landing/
node tools/audit.js                         # SEO + structure audit (0 errors = good)
node --test calculator-landing/calc.test.js # calculator unit tests (15)

cd deploy/movik-landing && vercel deploy --prod --yes
```

Full rebuild order: the three `build*` scripts → `deploy-staging` → `audit`.

Run `check` before every build — it is the safety net against shipping a
half-translated page. Current coverage: carriers 89/89, brokers 79/79,
calculator 60/60, and `audit.js` passes 0 errors / 0 warnings across all
10 pages.
